# 缺陷修复记录

本文档用于记录项目开发过程中发现的重要缺陷及其修复过程，以便后续查阅和经验总结。

## 1. 本地测试中 API Key 鉴权失败 (403 Forbidden)

### 问题描述
在本地执行 `npm test` 时，尽管已经在 `.dev.vars` 中配置了 `TASK_API_KEY=your_test_api_key_here` 且 `wrangler dev` 正常启动，但 API 测试始终返回 `403 Forbidden`。

### 根本原因
1. **环境变量优先级**：在当前 Shell 环境中，已经存在一个名为 `TASK_API_KEY` 的环境变量（包含一个随机生成的随机字符串）。
2. **测试脚本行为**：`tests/run_api_tests.js` 优先读取 `process.env.TASK_API_KEY`。由于 Shell 环境变量的优先级高于 `.dev.vars` 加载到测试进程中的默认逻辑（或者测试脚本直接读取了 Shell 环境），导致测试请求携带了错误的 Token。
3. **Wrangler 行为**：`wrangler dev --local` 会正确加载 `.dev.vars`，因此服务端期望的是 `your_test_api_key_here`，而客户端发送的是 Shell 中的随机字符串。

### 解决方案
在执行测试脚本时，显式指定与本地开发环境一致的 `TASK_API_KEY`：
```bash
TASK_API_KEY=your_test_api_key_here npm test
```

### 经验教训
- 在进行本地集成测试时，需特别注意当前 Shell 环境变量对测试脚本的影响。
- 建议在测试文档中明确说明环境变量的配置要求。

## 2. 周期性任务 ISO 时间格式解析失败

### 问题描述
当任务的 `due_date` 以 ISO 格式（如 `2026-03-05T08:00:00.000Z`）存储时，完成任务触发周期逻辑后，生成的下次截止日期变为 `null`。

### 根本原因
在 `src/worker/services/recurrence.ts` 中，解析日期字符串时盲目追加了 `"Z"` 后缀。对于已包含 `"Z"` 的 ISO 字符串，这会导致生成的字符串变为 `...ZZ`，使得 `new Date()` 无法解析并返回 `NaN`。

### 修复方案
在追加 `"Z"` 之前，先检查字符串是否已经以 `"Z"` 结尾：
```typescript
const date = new Date(currentDueDateStr.endsWith('Z') ? currentDueDateStr : currentDueDateStr + 'Z');
```

### 修复文件
- `src/worker/services/recurrence.ts`
