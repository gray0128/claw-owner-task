# 任务三：提取日期工具函数变更记录

## 背景
在 `src/worker/handlers/tasks.ts` 的顶部定义了 `isValidDate`、`normalizeDate`、`fromSqliteUtc` 等多个与日期时间格式化及转换相关的工具函数。这些函数逻辑通用，但被硬编码在该文件中，导致其它模块无法轻易复用这部分日期和时区解析相关逻辑。本任务的目标是将这些函数提取到专门的公共模块中进行管理，使得后续其它模块（如有需要）也可以调用。

## 变更详情

### 1. 新建 `helpers/date.ts` 工具模块
在 `src/worker/helpers/` 目录下创建了全新的 `date.ts` 模块：
- 将 `tasks.ts` 中的 `isValidDate`（ISO 日期正则校验）迁移至此。
- 将涉及基于运行时环境进行时区漂移修正的 `normalizeDate` 函数（核心用于将不同时区输入统一转换为 SQLite UTC 所需格式）迁移至此，并按需引入了 `utils.ts` 里的 `toSqliteUtc` 函数处理相关依赖。
- 将 `fromSqliteUtc` 函数（用于读取数据库出库数据并格式化成用户的当地时区响应结果）迁移至此。

### 2. 改造 `tasks.ts` 依赖引用
- 移除了原来在 `tasks.ts` 中前部的所有日期工具函数定义。
- 在头部采用 ES 模块化导入 `import { isValidDate, normalizeDate, fromSqliteUtc } from '../helpers/date';` 注入依赖，确保现有业务调用无需作任何修改。

### 3. 可靠性验证与自动化测试
- 运行 `npx tsc --noEmit` 成功通过类型校验，确认模块拆解未造成类型的错误引用或丢失。
- 解决了测试环境的上下文鉴权问题，并指定合规的测试密钥成功运行了 `npm test` 套件测试。
- 覆盖异常时间处理、单独更新 `due_date` 与 `remind_at`、以及带有各种时间格式的周期计算等核心自动化单元和边缘测试用例均完美通过，没有引入回归报错。
