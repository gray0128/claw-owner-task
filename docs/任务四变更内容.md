# 任务四变更内容：抽取 HTML 模板为共享组件

## 变更目标
根据重构优化计划，`summary.ts`、`share.ts` 和 `list.ts` 这三个公用页面处理器中存在大量重复的 HTML 页头结构、CSS 变量、重置基础样式、全局字体链接，以及重复定义的 `errorPage` 生成函数。本任务的目标是将这些重复的前端模板片段和错误展示页面抽取为共享组件，提升代码复用率和可维护性。

## 详细变更记录

### 1. 新增共享模板文件
为了统一管理公共样式和组件结构，在 `src/worker/templates` 目录下新增了以下两个文件：

- **`src/worker/templates/styles.ts`**
  - 定义了 `BASE_CSS_VARS`：提炼统一的 CSS 颜色、边距等全局视觉变量（`:root` 内部属性）。
  - 定义了 `BASE_STYLES`：提炼通用的重置样式和版式设置以及公共动画（如 `body` 的排版和 `fadeInUp` 动画）。
  - 定义了 `GOOGLE_FONTS_LINK`：提炼 Google Inter 字体的引入标签以统一排版风格。
  - 提供了 `materialIconsLink(iconNames: string[])` 辅助函数：用于按需生成引入 Material Symbols 图标包的标签代码。

- **`src/worker/templates/components.ts`**
  - 定义并导出了公用组件函数 `errorPageHtml(title: string, message: string)`，提供了一个标准且美观的前端错误/过期提示页面，替代了各自单独维护的逻辑。

### 2. 重构页面处理器逻辑
对负责渲染 HTML 响应的处理器执行了插值替换工作：

- **`src/worker/handlers/summary.ts`**
  - 移除了长串硬编码的错误页面字符串，统一通过调用 `errorPageHtml` 来渲染页面不存在（或 `uuid` 失效）与网页过期的响应（404 与 403 状态）。
  - 使用了导入的 `BASE_CSS_VARS`、`BASE_STYLES`、字体链接生成函数代替此前的冗长内联头部，极大精简了单文件的行数。
  - *附带优化*：由于 `npm test` 的编译校验暴露了遗留的构建问题，在本次重构中一并清理了文件中已经废弃的 `qqbot` 依赖报错片段。

- **`src/worker/handlers/share.ts`**
  - 删除了文件尾部的内部私有 `errorPage` 函数，改为调用共享模块实现代码复用。
  - 修改模板字符串中内联配置内容，利用 `materialIconsLink(['alarm', 'autorenew', 'bolt', ...])` 仅请求当前详情页实际需要的 Material 图标，降低重复成本。

- **`src/worker/handlers/list.ts`**
  - 同样删除了其内部冗余的 `errorPage` 函数。
  - 利用全局提供的常量和 CSS 变量进行结构重组，保证多任务列表与单任务展示拥有一脉相承的基础 UI 与字体样式。

## 验证与结论
- 已修复遗留的 TypeScript 校验报错，并通过 `npx tsc --noEmit`。
- API 测试套件所有校验指标均通过。所有涉及 Hono 原生 `html` 响应的功能路由（`/summary/:uuid`、`/share/:uuid`、`/list/:uuid`）等渲染结构均能正确反馈。
