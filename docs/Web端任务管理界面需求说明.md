# Web 端任务管理界面需求说明
## 文档版本：1.0.0
更新时间：2026-03-10

## 一、概述
本需求定义了 AI 端任务管理界面的功能设计，旨在为用户提供直观、易用的网页版任务管理操作入口，补充 CLI 与 API 功能，实现任务的全生命周期管理。界面采用极简设计理念，保持与现有系统架构完全兼容，同时新增 GitHub 账号绑定鉴权机制，替代原有 API Key 手动输入方式，提升用户体验与安全性。

## 二、核心设计原则
### 1. 极简体验
- 保持与现有系统的 AI 友好设计，所有操作均通过标准 API 接口实现，与 CLI 功能完全对齐
- 界面设计简洁高效，无多余功能克制，聚焦核心任务管理场景
- 响应式设计，支持桌面端与移动端自适应展示

### 2. 安全可靠
- 采用 GitHub OAuth 2.0 授权登录鉴权机制，替代手动输入 API Key 的方式，提高安全性
- 所有 API 请求自动携带授权信息，无敏感信息明文存储
- 权限校验与现有系统的`TASK_API_KEY`机制完全兼容，不改变后端鉴权逻辑

### 3. 功能对齐
- 所有功能与 CLI 保持一致，支持任务的增删改查、分类标签管理、提醒设置等核心能力
- 支持 Markdown 内容渲染，与系统现有格式完全兼容

## 三、功能需求
### 1. 账号与权限模块
#### 1.1 GitHub 登录
- 支持 GitHub 账号授权登录
- 登录流程：
  - 用户点击「GitHub 登录」按钮，跳转至 GitHub 授权页面
  - 授权成功后回调到 web 界面，后端验证用户身份
  - 后端绑定用户 GitHub ID 与系统`TASK_API_KEY`关联，自动获取对应 API Key 权限
  - 登录状态有效期 30 天，支持自动续期
  - 支持退出登录功能
- 未登录状态下，所有功能不可用，仅展示登录入口

#### 1.2 权限校验
- 所有 API 请求自动携带授权 Token，后端统一校验权限
- 权限不足时自动跳转至登录页面
- 支持多账号切换功能（可选）

### 2. 任务管理模块
#### 2.1 任务列表展示
- 列表字段：
  - 任务标题（高亮显示）
  - 任务描述（支持截断展示）
  - 优先级标识（高/中/低，不同颜色区分）
  - 状态标识（待处理/进行中/已完成/已取消）
  - 分类标签（彩色标签展示）
  - 截止时间/提醒时间（逾期标红提示）
  - 操作按钮（编辑/删除/完成/取消
- 列表支持分页或无限滚动加载
- 已完成任务默认折叠展示或灰色透明度降低

#### 2.2 任务筛选与搜索
- 状态筛选：全部/待处理/进行中/已完成/已取消
- 优先级筛选：全部/高/中/低
- 分类筛选：支持按分类筛选
- 标签筛选：支持按标签多筛选
- 搜索功能：支持标题和描述模糊搜索
- 排序功能：支持按创建时间/更新时间/截止时间/优先级排序

#### 2.3 任务添加功能
- 表单字段：
  - 任务标题（必填）
  - 任务描述（支持 Markdown 格式）
  - 优先级选择（高/中/低，默认中）
  - 状态选择（默认待处理）
  - 分类选择（可选）
  - 标签选择（支持多选，支持自动创建新标签）
  - 截止时间（可选，支持日期时间选择器）
  - 提醒时间（可选，支持日期时间选择器）
  - 重复规则（无/每天/每周/每月）
- 提交后自动刷新列表
- 支持表单重置功能

#### 2.4 任务编辑功能
- 点击任务编辑按钮，弹出编辑表单
- 表单字段与添加任务一致，自动填充原有数据
- 支持修改所有字段
- 提交后自动更新列表

#### 2.5 任务删除功能
- 点击删除按钮，弹出二次确认弹窗
- 删除成功后自动从列表移除

#### 2.6 任务状态变更
- 支持一键标记完成/取消完成
- 支持状态切换
- 周期任务完成后自动生成下一期任务

### 3. 分类与标签管理模块
#### 3.1 分类管理
- 分类列表展示（名称+颜色）
- 支持新增分类（名称+颜色选择）
- 支持编辑分类
- 支持删除分类（关联任务自动置空分类）

#### 3.2 标签管理
- 标签列表展示
- 支持新增标签
- 支持删除标签（关联任务自动移除该标签

### 4. 系统设置模块
- 基础信息展示：系统版本、当前登录用户信息
- 时区设置：与服务端`USER_TIMEZONE`保持一致
- 推送渠道设置：Bark/Telegram 推送开关配置
- API 信息展示：仅展示，不支持修改

### 5. 移动端适配
- 适配手机端界面自适应布局
- 触摸操作优化，按钮尺寸适配移动端操作
- 手势操作支持（左滑删除/完成）

## 四、界面设计规范
### 1. 设计风格：极简现代风格，与现有 CLI 输出风格保持一致
### 2. 配色方案：
- 主色调：蓝色系（#165DFF）
- 高优先级：红色系（#F53F3F）
- 中优先级：橙色系（#FF7D00）
- 低优先级：绿色系（#00B42A）
- 已完成：灰色系（#86909C）
- 逾期：红色高亮

### 3. 交互规范
- 所有操作均有 loading 状态提示
- 操作成功/失败均有 Toast 提示
- 表单校验提示明确的二次确认
- 加载失败有占位符
- 空状态有引导提示

## 五、技术实现要求
### 1. 前端技术栈
- 保持现有无构建方案：原生 HTML/CSS/JS，ES Modules
- 无需构建工具依赖，保持极简部署
- 兼容现代浏览器支持

### 2. 后端接口改造
- 新增 GitHub OAuth 授权相关接口：
  - `GET /api/auth/github/url`：获取 GitHub 授权链接
  - `GET /api/auth/github/callback`：授权回调处理
  - `GET /api/auth/userinfo`：获取当前登录用户信息
  - `POST /api/auth/logout`：退出登录
- 原有 API 接口保持不变，仅新增 Token 校验兼容 GitHub 授权与 API Key 两种鉴权方式

### 3. 数据存储
- 前端 Token 存储在 LocalStorage，有效期 30 天
- 无敏感信息明文存储

## 六、非功能需求
### 1. 性能要求：页面加载时间 < 2s
### 2. 兼容性：https 部署
### 3. 安全性：
- 防止 XSS 防护
- CSRF 防护
- 授权 Token 防窃取防护
- 输入内容 XSS 过滤

## 七、与现有系统兼容性
- 完全兼容现有 API 接口，与 CLI 功能完全对齐
- 不影响现有 CLI 用户使用
- 现有 API Key 鉴权方式保留，与 GitHub 鉴权并存
- 数据模型完全复用，无变更

## 八、开发优先级
1. 核心功能：GitHub 登录、任务列表、新增/编辑/删除
2. 次级功能：筛选搜索、分类标签管理
3. 扩展功能：移动端适配、系统设置
