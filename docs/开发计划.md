# 开发计划清单 (Development Roadmap)

本项目将按照以下四个阶段进行开发，每个阶段都包含明确的交付物和验收标准。

## 阶段 1：基础设施与后端核心 (Foundation & Backend)
**目标**：建立云端存储与核心 API 逻辑。

- [x] **1.1 项目初始化**
    - 初始化 `package.json` 及 TypeScript 配置。
    - 配置 `wrangler.toml`（D1 绑定、环境变量、Cron Triggers）。
- [x] **1.2 数据库 Schema 实现**
    - 编写迁移脚本 `src/worker/db/migrations/001_initial_schema.sql`。
    - 实现任务、分类、标签及关联表的创建。
- [x] **1.3 核心中间件开发**
    - `auth.ts`: 实现基于 Bearer Token 的强制鉴权。
    - `timezone.ts`: 实现基于 `USER_TIMEZONE` 的 I/O 时间转换逻辑。
- [x] **1.4 业务逻辑服务**
    - `bark.ts`: 集成 Bark 推送 API。
    - `recurrence.ts`: 实现每日/每周/每月周期任务的推算逻辑。
- [x] **1.5 API 路由实现**
    - `GET /api/info`: 实现 AI 自发现接口。
    - `GET/POST/PUT/DELETE /api/tasks`: 实现任务 CRUD（含语义搜索及元数据处理）。
    - `GET/POST/DELETE /api/categories & /api/tags`: 实现归类管理。
    - `POST /api/remind/check`: 实现双路径提醒触发逻辑。

## 阶段 2：跨平台 CLI 工具 (CLI Tooling)
**目标**：提供 AI Agent 和终端用户高效交互的界面。

- [x] **2.1 CLI 框架搭建**
    - 使用 `commander.js` 初始化 `src/cli/index.js`。
- [x] **2.2 API 客户端封装**
    - 实现带鉴权和错误拦截的请求工具。
    - 增加**离线感知**逻辑，捕获网络异常并友好提示。
- [x] **2.3 命令集实现**
    - 实现 `task add`, `task list`, `task update`, `task complete`, `task delete` 等基础命令，包含修改容错机制与时间参数 ISO 格式兼容（严格校验）。
    - 实现 `task info` (展示自发现数据)。
    - 实现无环境变量阻断的 `--help` 发现机制。
    - 实现 `task check --channel agent` 用于触发本地提醒。
- [x] **2.4 格式化输出**
    - 针对人类用户提供表格化输出。
    - 针对 AI 提供全局 `--json` 结构化输出（包含标准化错误返回）。
- [x] **2.5 高性能二进制实现 (Rust)**
    - 使用 Rust 重写 CLI，位于 `cli-rust/` 目录。
    - 通过 `cargo build --release` 编译原生二进制。
    - 功能与 Node.js 版完全一致，零依赖分发。

## 阶段 3：原生 Web 界面 (Native Web UI)
**目标**：提供无需构建的、极简的任务管理后台。

- [x] **3.1 基础脚手架**
    - 创建 `src/web/index.html` 及 `css/style.css`。
- [x] **3.2 原生 ES Modules 开发**
    - `api.js`: 封装浏览器端 API 调用。
    - `ui.js`: 实现基于原生 DOM 的列表渲染。
- [x] **3.3 功能实现**
    - 任务列表展示（支持按状态/标签筛选）。
    - 快速创建与标记完成。
    - 设置页面（配置 API Key 和时区）。

## 阶段 4：集成、测试与交付 (Integration & QA)
**目标**：验证全链路闭环，完成交付。

- [x] **4.1 数据 Mock 与冒烟测试**
    - 导入 `tests/mock_data.sql` 进行压力与边界测试。
    - 运行 `tests/api_tests.http` 验证 API 正确性。
- [x] **4.2 OpenCLaw 集成验证**
    - 配置 OpenCLaw Cron 定时执行 `task check`。
    - 验证 macOS/Linux 环境下的通知触发。
- [x] **4.3 最终文档同步**
    - 更新 `CLAUDE.md` 记录常用命令。
    - 完成交付演示。

## 阶段 5：多模态与多渠道扩展 (Extensions & Integrations)
**目标**：接入外部 IM 平台，实现跨渠道的任务触达与对话式管理。

- [x] **5.1 Telegram 机器人接入与提醒渠道扩展**
    - 增加 `TELEGRAM_BOT_TOKEN` 和 `TELEGRAM_CHAT_ID` 环境变量。
    - 实现 `src/worker/services/telegram.ts`，封装 Telegram API 进行消息下发。
    - 升级 `handlers/remind.ts` 提醒逻辑，支持配置化推送到 Telegram 渠道。
- [x] **5.2 Telegram 对话式任务管理**
    - 构建 Webhook 接收端点，安全接收来自 Telegram 用户的交互消息。
    - 接入后端的 AI 处理主链路，将 Telegram 输入转换为等效的 `claw-task ai` 指令。
    - 实现输出适配器：针对聊天窗口优化格式排版验证（如 Markdown 表格、加粗高亮等），提升交互体验。
- [x] **5.3 Telegram 斜杠命令支持 (Slash Commands)**
    - 增加 `/summary` (/总结) 命令拦截，直接触发总结生成。
    - 增加 `/add` (/添加) 命令拦截，实现非 AI 解析的任务快速录入。
- [x] **5.4 单条任务独立查看页面 (Task Share Page)**
    - 实现 `task_shares` 表及基于 UUID 的时效性分享机制。
    - 在 API 响应中动态注入 `view_url` 字段。
    - 实现公共 HTML 渲染页面，适配移动端查看。
