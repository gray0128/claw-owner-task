# 测试计划与用例 (v1.1.1)

本计划用于验证 `claw-owner-task` 的核心逻辑，确保其符合 AI 友好性、跨平台兼容性及多渠道提醒的设计目标。

## 1. 基础功能测试 (CRUD)

| ID | 测试场景 | 预期结果 |
|---|---|---|
| F-01 | 创建任务（基本信息与 ISO 时间格式） | 任务保存成功，时间成功被格式化及解析，默认状态为 `pending`，优先级为 `medium` |
| F-02 | 创建任务（关联分类与自动创建标签） | 任务与分类/标签关联正确；支持传入标签名称，对于不存在的合法标签名称自动创建并关联 |
| F-03 | 更新任务状态至 `completed` | 状态变更成功，更新 `updated_at`；若有周期规则，自动推算下次时间并重置为 `pending` |
| F-04 | 删除任务测试 | 可以通过 `delete` 命令正确删除指定任务 |
| F-05 | 删除分类/标签 | 分类删除应提示或处理关联任务；标签删除应级联删除 `task_tags` 关联 |
| F-06 | 修改任务（更新各个字段） | 能够成功更新任务标题、状态、优先级、分类、标签（更新关联）、时间等，更新 `updated_at` |
| F-07 | 任务列表多维度筛选 | 能根据 `status`, `priority`, `category_id`, `due_date`, `remind_at`, `tag` (ID) 及 `tag_name` 准确过滤任务列表 |
| F-08 | 标签格式校验 | 创建标签或按名称关联标签时，非中英文和数字的标签名（如包含特殊字符或空格）应被拒绝 |

## 2. AI 友好性测试 (AI-First)

| ID | 测试场景 | 预期结果 |
|---|---|---|
| AI-01 | 调用 `/api/info` | 返回系统版本、所有可选状态、分类、标签及当前 `USER_TIMEZONE` |
| AI-02 | 存储并读取 `metadata` | 存入 JSON 字符串（如对话 ID），读取时能正确解析并返回 |
| AI-03 | 溯源验证 (`source`) | 明确标记任务是由 `openclaw` 或 `user` 创建 |
| AI-04 | 语义化搜索 (`q`) | 输入标题或描述中的关键词，API 应返回匹配的任务列表 |
| AI-05 | 无环境变量阻断的 `--help` 自发现 | 未设置 `TASK_API_KEY` 时运行 `--help` 应正常返回命令列表而不报错退出 |
| AI-06 | 机器友好输出与错误结构 (`--json`) | 在任何包含表格输出的命令后加上 `--json` 应返回格式化的 JSON 字符串；错误输出采用标准化 JSON 结构 |

## 3. 提醒机制测试 (Dual-Channel)

| ID | 测试场景 | 预期结果 |
|---|---|---|
| R-01 | 检查云端渠道 (`channel=cloud`) | Worker 触发 Bark 推送，更新 `reminded` 标记，API 返回成功 |
| R-02 | 检查 Agent 渠道 (`channel=agent`) | 返回当前一分钟内到期的任务 JSON 列表，**不**触发 Bark，不更新标记（由 Agent 确认） |
| R-03 | 重复提醒阻止 | 已标记为 `reminded` 的任务不再出现在提醒检查结果中 |

## 4. 周期逻辑测试 (Recurrence)

| ID | 测试场景 | 预期结果 |
|---|---|---|
| REC-01 | 完成每日任务 (`daily`) | 状态重置为 `pending`，`due_date` 自动推后 24 小时 |
| REC-02 | 完成每周任务 (`weekly`) | 状态重置为 `pending`，`due_date` 自动推后 7 天 |

## 5. 安全与时区测试

| ID | 测试场景 | 预期结果 |
|---|---|---|
| S-01 | 无鉴权请求 | Header 缺少 API Key 时返回 `401 Unauthorized` |
| T-01 | 时区偏移验证 | 设置 `USER_TIMEZONE=UTC` 和 `Asia/Shanghai` 分别获取同一任务，显示时间相差 8 小时 |
| T-02 | 时间格式严格校验 | 传入非法格式的 `due_date` 或 `remind_at` 时 API 应当拒绝并抛出明确错误 |
